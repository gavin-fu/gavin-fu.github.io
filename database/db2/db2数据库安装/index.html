表空间注意事项

  1. 表空间类型选择（SYSTEM / DATABASE）

  2. 选择合适的页大小(PAGESIZE)

  3. 扩展数据块大小（EXTENTSIZE）

  4. 预读大小选择（PREFECHSIZE）
    PREFECHSIZE = 表空间容器设备数 * 扩展数据块带小

  5. 文件系统和裸设备（CIO/DIO）

  6. 设置 容器开销 和 传送速率（OVERHEAD / TRANSFERRATE）
    这两个参数用于确定查询优化期间的 I/O 成本，是所有容器的平均值，单位毫秒。

    • Overhead = 平均寻道时间 + (((1 / 磁盘转速) 60 1000) / 2)
    • Transferrate = (1 / 传送速率) 1000 / 1024000 4096(假设用4K页)

表空间设计原则

  1. 对于 OLAP分析型应用，以连续大片读为主，采用较大的页面，降低 I/O请求次数；
  2. 选择 DMS大型表空间，可以灵活定义；
  3. 设置预读大小：可以在创建表空间的时候，将 PrefetchSize设置为 Automatic，预读操作在查询使用所需的数据之前读入这些数据，
    因为这些数据已经在内容中了，这样就不必等待执行 I/O了；
  4. 数据和索引使用不同的表空间；
  5. 每个表空间使用独立的缓冲池；

缓冲池

  1. 每个表空间都会与特定的缓冲池相关；
  2. 想要表空间与缓冲池相关，该缓冲池与表空间必须具有相同的页大小；
  3. 表空间与缓冲池关联是在使用 CREATE TABLESPACE 语句创建表空间时定义的，但是可以使用 ALTER TABLESPACE 语句更改关联。
